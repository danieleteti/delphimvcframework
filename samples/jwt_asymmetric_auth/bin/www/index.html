How RSA JWT differs from HMAC JWT

HMAC (HS256): One shared secret for both signing and verification.
RSA (RS256): Private key signs, public key verifies. The public key can be shared freely.

Private Key
signs tokens
→
JWT Token

→

verifies tokens

In a microservice architecture, only the auth server needs the private key. All other services verify tokens using the public key.

Key Difference: HS256 vs RS256 in Delphi

      
// HS256 (symmetric - shared secret):
Engine.AddMiddleware(TMVCJWTAuthenticationMiddleware.Create(
  AuthHandler, ConfigClaims,
  'my_shared_secret',       // <-- same secret for sign + verify
  '/api/login'));

// RS256 (asymmetric - key pair):
uses MVCFramework.JWT.RSA;   // <-- requires TaurusTLS (OpenSSL 1.1.1+)

var LSigner := TRSAJWTSigner.CreateFromFiles(
  JWT_RS256,                // <-- algorithm
  'keys/private.pem',      // <-- private key (sign)
  'keys/public.pem');      // <-- public key (verify)

Engine.AddMiddleware(TMVCJWTAuthenticationMiddleware.Create(
  AuthHandler, ConfigClaims,
  LSigner,                  // <-- signer instead of secret
  '/api/login'));
