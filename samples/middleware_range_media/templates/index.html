How It Works

The TMVCRangeMediaMiddleware intercepts requests matching a URL prefix and serves files with HTTP Range support (RFC 7233):

  • GET /media/sample.mp3 without Range header → 200 OK (full file)
  • GET /media/sample.mp3 with Range: bytes=0-65535 → 206 Partial Content
  • Invalid range → 416 Range Not Satisfiable

Directory traversal is blocked. Only files within the configured document root are accessible.

Test with curl

Full file:

curl -v http://localhost:8080/media/sample.mp3

Range request (first 64 KB):

curl -v -H "Range: bytes=0-65535" http://localhost:8080/media/sample.mp3
