How HTTP-Only Cookies Work

Classic HTTP Requests (Browser Navigation):

  • When you click a link or submit a form, the browser automatically includes all cookies for that domain
  • No JavaScript involved - the cookie is sent in the Cookie HTTP header
  • Works transparently for server-rendered pages, redirects, and traditional web apps
  • Example: <a href="/admin/profile">My Profile</a> - cookie sent automatically

JavaScript (fetch/XHR):

  • You cannot read the cookie value with document.cookie (HttpOnly protection against XSS)
  • But the browser still sends it automatically if you use credentials: 'same-origin' or 'include'
  • Example: fetch('/api/data', { credentials: 'same-origin' })
  • For cross-origin requests, use credentials: 'include' and configure CORS on the server

HTMX:

  • HTMX uses XMLHttpRequest internally, which sends cookies automatically for same-origin requests
  • No configuration needed! Just use HTMX normally: <button hx-get="/admin/profile">Load</button>
  • For cross-origin, add hx-request='{"credentials": "include"}' to your elements
  • Perfect for server-rendered HTML fragments with authenticated endpoints

5. Verify Cookies with DevTools

Chrome / Edge

  1. Press F12 to open DevTools
  2. Go to Application tab
  3. Expand Storage > Cookies
  4. Click on localhost
  5. Find jwt_token cookie

Firefox

  1. Press F12 to open DevTools
  2. Go to Storage tab
  3. Expand Cookies
  4. Click on your domain
  5. Check cookie attributes
What to verify:
  • HttpOnly: Should show checkmark (prevents JavaScript access)
  • SameSite: Should show "Strict" (CSRF protection)
  • Secure: Shows checkmark in production HTTPS (disabled for local HTTP)
  • Path: Should be "/" (cookie sent for all paths)
Security Note: In production, always use HTTPS! The cookie will have Secure=true which requires HTTPS. This demo uses HTTP for local development.
